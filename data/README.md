# Data layout (populate & ingest)

Keep **real exports** out of Git — use **`data/local/`** (gitignored) for private dumps.

## 1. Optional structured JSON → `knowledge_items`

Use **`npm run ingest:seed`** with parser output or a hand-built JSON array (see **`scripts/ingest/seed-posts.ts`**). Shape matches **`docs/PROGRESS.md`** (`category`, `action_items`, `source_type`, etc.).

## 2a. 小红书-style CN posts → structured JSON (rule-based)

| Location | Purpose |
|----------|---------|
| **`data/raw_posts/*.txt`** | Optional parser inputs — one post per `.txt`. See **`data/raw_posts/README.txt`**. |
| **`data/local/raw_posts/`** (optional) | Same as above but **private** — folder is gitignored; run parser with `-i path`. |
| **`data/output/seed_posts.json`** | Generated by the parser (**gitignored** — run locally). |

```bash
python3 scripts/ingest/parse_cn_us_posts.py
# or: npm run ingest:parse-cn
python3 scripts/ingest/parse_cn_us_posts.py --input data/raw_posts --output data/output/seed_posts.json
```

**Populate Supabase (after `seed_posts.json` exists):**

```bash
npm run ingest:seed -- --mode db
# or via API (MiniMax re-extracts each row): npm run ingest:seed -- --mode api --delay-ms 1200
```

## 2b. WeFlow (optional) — chat export → this repo

[WeFlow](https://github.com/hicccc77/WeFlow) exports WeChat chats to **`.txt` / `.json`**. After export, use **`npm run ingest:local`** on a path (often under **`data/local/`**). See previous revision history in git for longer WeFlow notes if needed.

**Do not** commit raw chat exports to Git.

## 3. Local-only (never commit)

**`data/local/`** — WeChat / TikTok unzips, personal exports, parser output. Everything here is **gitignored**.

## 4. Scripts

| Script | Role |
|--------|------|
| **`scripts/bulk-ingest-local.ts`** | Walks folders of `.txt` / `.md` / `.log` and POSTs chunks to **`POST /api/message`**. |
| **`scripts/ingest/parse_cn_us_posts.py`** | **`data/raw_posts/`** → **`data/output/seed_posts.json`**. |

Prefer user **export** or **paste** in **`/connect`**, **Photon** for iMessage (macOS), not live scraping.

## Quick commands

```bash
# API running: npm run dev
USER_ID=<uuid> npm run ingest:local -- "/path/to/folder-of-txt"
```
